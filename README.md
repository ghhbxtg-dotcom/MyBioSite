# deepsy-bio

Личная страница-визитка с встроенным чатом и прямой связью через Telegram-бота. Всё в одном HTML-файле, бэкенд на Flask.

Собирал под себя, но архитектура простая, можно взять за основу и переделать под свои нужды.

##Что внутри

Сайт состоит из нескольких секций: главный экран, обо мне, проекты, контакты и FAQ. Всё на одной странице, скролл с плавной анимацией появления блоков.

Главная фишка - встроенный чат. Работает так: пользователь открывает попап, выбирает тему вопроса из готовых вариантов (заказ, сроки, портфолио, контакты), получает заготовленный ответ. Если не нашёл нужного — пишет напрямую, сообщение летит на сервер и пересылается в Telegram владельцу. Ответ из Telegram тоже попадает обратно в чат - сайт опрашивает сервер каждые 2 секунды.

Бэкенд минимальный: два эндпоинта, Flask, хранит последний ответ в памяти. Работает на Render, но подойдёт любой VPS или Railway.

## Стек

- HTML/CSS/JavaScript - никаких фреймворков, чистый ванила
- Python + Flask - бэкенд для чата
- Telegram Bot API - пересылка сообщений
- Деплой фронта - статика (GitHub Pages, Netlify, любой хостинг)
- Деплой бэка - Render (бесплатный tier)

Шрифты: Inter, JetBrains Mono, Space Grotesk - подключены через Google Fonts.

## Как запустить локально

```bash
# клонируй репо
git clone https://github.com/ghhbxtg-dotcom/MyBioSite
cd MyBioSite

# установи зависимости для сервера
pip install flask requests

# создай .env или пропиши токены прямо в server.py
TELEGRAM_BOT_TOKEN=твой_токен
TELEGRAM_CHAT_ID=твой_chat_id

# запусти сервер
python server.py

# открой index.html в браузере
# или запусти через live server в VSCode
```

Без бэкенда сайт тоже работает - просто чат не будет отправлять сообщения. Все остальные секции статические.
## Как устроен чат
Логика разбита на две части.
**Фронт** (`index.html`):
- При открытии чата показывает приветственное сообщение и 4 кнопки с темами
- При выборе темы - отображает заранее написанный ответ
- Если пользователь хочет написать напрямую - отправляет POST на `/send`
- Каждые 2 секунды делает GET на `/getReply` и отображает ответ, если он появился
**Бэк** (`server.py`):
- `POST /send` - принимает сообщение, пересылает его в Telegram через Bot API
- `GET /getReply` - отдаёт последний ответ, сохранённый в памяти
- Входящие сообщения от владельца в Телеграм -> вебхук или polling -> сохраняются и отдаются фронту
Это не realtime в полном смысле, но для личного сайта - достаточно. Задержка максимум 2 секунды.
## Кастомный курсор
На десктопе курсор полностью кастомный: три слоя - точка (мгновенно), кольцо (с лагом через lerp), свечение (blur + mix-blend-mode: screen). Меняет форму в зависимости от того, на что наведён — интерактивный элемент, текстовое поле или заголовок.
На мобилках и устройствах без точного указателя курсор автоматически скрывается — определяется через `matchMedia('(hover: hover) and (pointer: fine)')`.

## Анимации и детали
- Фоновая сетка с drift-анимацией через CSS `background-size` + `@keyframes`
- Scroll progress bar наверху страницы
- Reveal-анимации элементов при скролле - `IntersectionObserver`, порог 25%
- Карусель для проектов со свайпом на тач-устройствах
- Боковое меню с spring-анимацией (`cubic-bezier(0.34, 1.56, 0.64, 1)`)
- Все ховеры, переходы и состояния прописаны вручную

## Деплой
**Фронт** - любой статический хостинг. Проще всего GitHub Pages:
```bash
# пуш в репо, включи Pages в настройках
```
**Бэк** — Render, Railway, или любой VPS с Python. На Render работает на бесплатном плане, но засыпает через 15 минут без запросов — первый запрос после сна будет с задержкой.
В `index.html` адрес бэка определяется так:
```js
const API_BASE =
    window.location.hostname === "localhost"
        ? "http://127.0.0.1:5000"
        : "https://твой-сервер.onrender.com";
```

## Что можно поменять под себя

- Текст в секциях - всё прямо в HTML, ищи по комментариям
- Темы чата - объект `chatTopics` в `<script>`, добавляй/убирай ключи
- Ссылки на контакты - секция `.links` в HTML
- Цвета - CSS-переменные в `:root`
- Шрифты - заменяются в одном месте в `<head>`
- Секцию проектов можно раскомментировать (она скрыта через `display: none !important` в конце стилей)
